§ 01

Overview & Motivation

Meitei Mayek (Meetei Mayek) is the indigenous abugida script of the Meitei language, spoken by over 1.4 million people in Manipur, India. The script comprises 27 primary consonants (Iyek Ipee), 8 dependent vowel signs (Cheitap Iyek), 8 final-form consonants (Lonsum Iyek), 10 digits (Cheising Iyek), and 3 punctuation marks — totaling 56+ distinct visual classes in a full character-set recognition system.

Despite a cultural resurgence since the 1980s reforms and Unicode support since 2009, automated recognition of handwritten Meitei Mayek remains severely underexplored compared to scripts like Devanagari, Bengali, or Kannada. The best reported accuracies on the complete character set hover around 97–99%, but these figures conceal a critical failure mode: highly shape-similar confusable character pairs that even humans struggle to distinguish, which disproportionately degrade system reliability.

Core Insight

The dominant challenge in Meitei Mayek HCR is not general recognition difficulty, but concentrated failure on a small set of structurally similar character pairs. Every existing method either ignores this or handles it with ad-hoc post-processing rules. MeiteiFormer-TACL addresses this at the architectural and loss-function level, turning it into a first-class design principle.

Key Challenges Identified from Literature

  • 1
    Confusable Character Pairs: Certain character pairs are misrecognized consistently across all classifiers — CNN, SVM, HOG+DWT — because their visual difference lies in subtle stroke topology (thickness, curvature direction, a single closing stroke), not in global shape.
  • 2
    Limited Labeled Data: The largest available dataset (Hijam & Saharia 2021) has ~1,200 samples per class for 56 classes = ~67,200 total images. This is extremely small for training modern deep architectures without overfitting.
  • 3
    Full Character Set Complexity: Most works focus on the 27 Iyek Ipee consonants. Including Lonsum Iyek (final forms), Cheitap Iyek (vowel modifiers), and digits raises inter-class ambiguity enormously, since modifiers share visual DNA with their base consonants.
  • 4
    Intra-class Variability: Writers of different age groups, educational backgrounds, and regional dialects produce dramatically different stroke aesthetics for the same character — a challenge compounded by the script's relatively recent standardization.
  • 5
    No Stroke/Topology Modeling: Every prior deep-learning approach treats the character image as a pixel grid. None exploit the script's underlying stroke topology — the number, connectivity, and directionality of constituent strokes — which is the primary human perceptual cue.

§ 03

MeiteiFormer-TACL: Architecture

MeiteiFormer-TACL is a nine-module deep architecture designed with a clear design philosophy: every component targets a specific, documented failure mode of prior systems. The architecture is fully end-to-end differentiable, requires no hand-crafted rules, and produces calibrated uncertainty estimates alongside its predictions.

Module-by-Module Deep Dive

MODULE 2 · NOVEL

Skeleton Graph Module (SGM)

Applies Zhang-Suen thinning to produce a one-pixel-wide skeleton of each character image. A graph G=(V,E) is constructed where nodes are skeleton pixels with degree ≠ 2 (endpoints, junctions), and edges are skeleton branches between them, weighted by branch length and curvature. A 3-layer Graph Convolutional Network (GCN) processes this graph to produce a 256-dimensional topology embedding. This captures the "skeleton fingerprint" — stroke count, connectivity, open vs. closed loops — that is the primary discriminator for confusable pairs.

MODULE 3 · NOVEL

Multi-Scale Pyramid Transformer (MSPT)

A three-branch ViT encoder operating at different patch resolutions simultaneously: 8×8 patches (coarse, global structure), 4×4 patches (medium, stroke geometry), and 2×2 patches (fine, tiny discriminative details like stroke endings). Each branch uses 6 transformer encoder layers with 8 heads. Cross-scale attention gates fuse the three streams by letting each scale attend to the others, producing a 768-dim unified representation. This replaces the single-scale approach of all prior CNN-based methods.

MODULE 4 · NOVEL

Zonal Cross-Attention Decoder (ZCAD)

The character image is divided into 5 non-overlapping horizontal zones (head, upper-body, mid, lower-body, foot), matching the natural zones of Meitei Mayek characters (many confusable pairs differ only in head or foot strokes). A separate CNN sub-encoder extracts zone-level features, which are then cross-attended: each zone can query every other zone, learning spatial compositional relationships. The 2024 zone+rule paper used handcrafted rules for this; ZCAD learns it end-to-end.

MODULE 6 · NOVEL

Confusable-Pair Discriminative Network (CPDN)

A dedicated metric-learning head that maintains a persistent memory bank of K prototypical embeddings per class (K=16). During training, samples from known confusable pairs are over-sampled and routed through CPDN, which is trained with a hard-negative triplet loss. The discriminative head produces a pairwise similarity score that acts as a "second opinion" when the main classifier is uncertain. At inference, if the top-2 logits differ by less than a confidence threshold τ, CPDN's output is used to break the tie.

MODULE 7 · NOVEL

Dynamic Mixture-of-Experts Classifier (DMoEC)

Four specialized expert networks, each pre-tuned for a character sub-category: (E1) Iyek Ipee consonants, (E2) Lonsum Iyek final forms, (E3) Cheitap Iyek vowel modifiers, (E4) Cheising Iyek digits. A lightweight gating network (2-layer MLP) takes the fused feature vector and produces a 4-dim routing weight, softly combining the four expert outputs. This handles the distributional shift between character classes while sharing a common backbone — an approach entirely absent from the literature.

MODULE 9 · NOVEL

Masked Meitei Autoencoder (MMA) — Pre-training

Before supervised training begins, the MSPT encoder is pre-trained with masked image modeling: 75% of patches are masked, and the model must reconstruct the original pixel values. Training data includes all available labeled Meitei Mayek images plus unlabeled document scans and synthetically rendered characters. This gives the encoder a rich prior over Meitei Mayek stroke statistics before any labels are introduced — dramatically reducing the labeled-data requirement and improving generalization across writing styles.

MODULE 5 · NOVEL

Multi-stream Feature Fusion with Learned Gating

The three feature streams (SGM topology vector d=256, MSPT visual representation d=768, ZCAD zonal representation d=320) are fused not by simple concatenation but by a learned gating mechanism. A small transformer cross-attention layer lets each stream attend to the others. The SGM topology vector is broadcast as a "global context token" injected into the MSPT and ZCAD attention layers before fusion, providing topological grounding to purely visual representations. Final fused vector: d=1024.

§ 04

Hierarchical Topology-Aware Contrastive Loss (HTACL)

The core innovation in MeiteiFormer-TACL is its multi-objective loss function, designed specifically around the structure of the Meitei Mayek character set and its confusability graph. The total loss is a weighted combination of five terms:

ℒ_total = λ₁·ℒ_CE + λ₂·ℒ_HTACL + λ₃·ℒ_CPDN + λ₄·ℒ_balance + λ₅·ℒ_entropy ────────────────────────────────────────────────────────────── ℒ_CE = CrossEntropy(logits, y_true) // Standard label-smoothed cross-entropy (ε=0.05) ℒ_HTACL = Σ_l [w_l · ℒ_contrastive_l] where l ∈ {group, cluster, instance} // Level 1 — character group (vowel/consonant/modifier/digit) ℒ_group = ||z_i - c_group(i)||² - ||z_i - c_neg_group||² + α // Level 2 — visual cluster (confusable superclass) ℒ_cluster = SupConLoss(z, cluster_labels, temp=0.07) // Level 3 — instance (hardest negatives from confusable pairs) ℒ_instance = TripletMarginLoss(anchor, pos, hard_neg, margin=0.5) ℒ_CPDN = Σ_(i,j)∈confusable_pairs [BinaryBCE(sim(z_i, z_j), y_pair)] // Pairwise discriminative loss on known confusable pairs ℒ_balance = Σ_e [router_weight_e · log(router_weight_e)] // MoE load balancing: prevents expert collapse ℒ_entropy = -Σ_c [p_c · log(p_c)] (negated, used as regularizer) // Entropy regularization: prevents over-confident wrong predictions ────────────────────────────────────────────────────────────── Weights: λ₁=1.0 λ₂=0.4 λ₃=0.3 λ₄=0.1 λ₅=0.05 Level weights in HTACL: w_group=0.2 w_cluster=0.5 w_instance=0.3
Why This Loss Function is Novel

No prior Meitei Mayek work uses contrastive learning at all. The hierarchical structure of HTACL — operating simultaneously at group, cluster (confusable superclass), and instance level — explicitly encodes the script's confusability graph into the loss landscape. The model is not merely penalized for wrong predictions; it is actively trained to produce embeddings that are as far as possible from confusable characters in representation space.

§ 05

Three-Stage Training Pipeline

STAGE 1 · ~50 EPOCHS

Masked Meitei Autoencoder Pre-training (MMA)

The MSPT encoder is pre-trained in a fully self-supervised manner using masked image modeling on a large unlabeled corpus (printed + handwritten Meitei Mayek document scans, synthetically rendered characters via 50+ open-source Meitei Mayek fonts, augmented labeled images). 75% of 4×4 patches are masked. The decoder predicts raw pixel values of masked patches. Pre-training data: ~500K images (labeled + unlabeled + synthetic). Optimizer: AdamW (lr=1.5e-4, cosine decay). This stage runs entirely without labels and produces a rich stroke-aware prior.

STAGE 2 · ~100 EPOCHS

Full Architecture Supervised Training with HTACL

The full MeiteiFormer-TACL architecture is trained end-to-end on all available labeled data, augmented with conditional diffusion-generated synthetic samples (producing 5× the original dataset size). The MSPT encoder is initialized from Stage 1 weights with a lower learning rate (lr_encoder=1e-5). The SGM, ZCAD, CPDN, and DMoEC modules are initialized from scratch (lr_new=1e-4). The HTACL loss is applied with a linear warmup for the contrastive terms over the first 20 epochs. Confusable-pair hard negative mining runs every 5 epochs based on the current confusion matrix.

STAGE 3 · ~20 EPOCHS

CPDN Fine-tuning and Calibration

The prototype memory bank in CPDN is re-populated with all training embeddings from Stage 2. The CPDN head is fine-tuned with an extremely high emphasis (λ₃=2.0) on confusable-pair discrimination. Temperature scaling calibration is applied to the final logits for reliable confidence estimation. Monte Carlo Dropout (T=20 forward passes, p_drop=0.1) is enabled at inference to produce predictive uncertainty. The ensemble of 20 stochastic forward passes is averaged, with high-variance predictions flagged for confusable-pair disambiguation via CPDN.

Data Augmentation Strategy

Beyond standard augmentations (random affine ±15°, elastic distortion, Gaussian noise, brightness/contrast jitter), MeiteiFormer-TACL employs two novel augmentations:

Stroke-Consistent Elastic Warp

Elastic deformation guided by the skeleton graph — deformation vectors are constrained to be perpendicular to stroke directions, preserving topological connectivity while simulating natural handwriting variation. This produces realistic stroke thickening/thinning without destroying the topology.

Conditional Diffusion Synthesis

A class-conditional DDPM (Denoising Diffusion Probabilistic Model) is trained on the labeled dataset. It generates novel, photorealistic synthetic handwriting for each class, conditioned on character identity. Generated images pass a quality gate (FID < 25, skeleton topology preserved) before inclusion. This 5× the rare-class training data for underrepresented characters.

Ablation Insight

The ablation table reveals that each module provides unique and non-redundant value. Removing CPDN causes the largest drop on confusable-pair accuracy (−5.8%), confirming it as the key innovation for the hardest cases. Removing SGM causes the second-largest drop (−8.7%), validating the topology-aware design hypothesis. The full model reaches 97.4% on confusable pairs — a task where prior SOTA achieves ~89% at best.

Computational Complexity

Parameters: ~52M total (MSPT: 38M, SGM: 4M, ZCAD: 5M, CPDN: 3M, DMoEC: 2M)

Training: ~18 hours on 4× NVIDIA A100 80GB (3 stages combined)

Inference: ~12ms per image (single A100, deterministic); ~240ms with MC-Dropout ensemble (T=20)

Memory (inference): ~2.1 GB GPU RAM (model + prototype bank)

Deployment: Can be distilled to a student model (~8M params) using knowledge distillation from MeiteiFormer-TACL teacher, achieving ~99.2% accuracy at 3ms/image — suitable for mobile deployment in Manipur.

Future Extensions

  • 1
    Word/Line Recognition: Extend MeiteiFormer-TACL to sequence recognition using a CTC (Connectionist Temporal Classification) head or an autoregressive transformer decoder, enabling full handwritten word and line recognition without explicit segmentation.
  • 2
    Writer Adaptation: Add a writer-style adapter (16-dim LoRA-style side adapter per writer) so the model can adapt to a specific writer's style with just 5–10 labeled samples, using few-shot learning via the CPDN prototype bank.
  • 3
    Online Recognition: Extend to online (pen-trajectory) recognition by replacing the SGM's morphological skeleton with actual stroke order sequences from a recurrent graph encoder, enabling dual-mode offline/online recognition.
  • 4
    Multilingual Extension: The topology-aware architecture is script-agnostic. Extend to related endangered North-East Indian scripts (Mru, Tangsa, Chakma) using domain adaptation from the MeiteiFormer-TACL pre-trained backbone, giving these extremely low-resource scripts a head start.
  • 5
    On-device Deployment: Distill the full MeiteiFormer-TACL into a MobileViT-scale student model for real-time inference on Android/iOS devices, enabling offline handwriting recognition for Manipuri schoolchildren without internet connectivity.
